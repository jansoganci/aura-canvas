# Aura UX Review Skill

Review and improve user experience for the Aura Canvas app.

## When to Use
- When adding new UI components
- When refactoring user flows
- When optimizing for mobile
- When improving accessibility

## Review Checklist

### Visual Hierarchy
- [ ] Clear focal points for each view
- [ ] Consistent spacing (Tailwind: p-4, p-6, gap-4, etc.)
- [ ] Proper typography scale
- [ ] Aura colors are prominent and distinguishable

### User Flow
Current flow: Landing → Upload → Edit → Display (Vote/Comment)

- [ ] Each step is clear
- [ ] Progress indication
- [ ] Easy to go back
- [ ] Clear CTAs

### Mobile Experience
- [ ] Touch targets min 44px
- [ ] Readable text without zoom
- [ ] No horizontal scroll
- [ ] Fast load time
- [ ] Thumb-friendly button placement

### Aura Color UX
- [ ] All 8 colors visually distinct
- [ ] Color meanings easily accessible
- [ ] Voting is intuitive
- [ ] Results are satisfying to view

### Accessibility
- [ ] Sufficient color contrast
- [ ] Alt text for images
- [ ] Keyboard navigation
- [ ] Screen reader friendly
- [ ] Focus indicators

### Performance
- [ ] Images optimized
- [ ] Loading states clear
- [ ] Error states helpful
- [ ] Smooth animations

## Common UX Improvements

### Quick Wins
1. Add loading skeletons instead of spinners
2. Animate vote count changes
3. Confetti on voting
4. Share button for results
5. Haptic feedback on mobile

### Engagement Features
1. "Your aura is..." reveal animation
2. Compare with friends
3. Aura history/timeline
4. Daily aura check-in

### Social Proof
1. Total votes count
2. "X people voted Purple"
3. Recent activity feed
4. Trending aura colors
